React脚手架使用addLessLoader导致less中的url路径错误

发表于 2020-07-14 更新于 2020-07-15
本文字数： 5.4k 阅读时长 ≈ 5 分钟

前言

上文中描述了使用react-app-rewired搭配customize-cra在React脚手架中使用less，多次尝试发现在调试环境中可以正常使用less且正确加载less中的url图片，但是在生产环境下，url就成了绝对路径导致图片没法展示出来，问题困扰几个月有余，今日终得解。

一、尝试

为了找出这个问题的所在，经过了如下的尝试

1.生产环境和开发环境的对比

开发环境可以加载出正确的图片

生产环境图片显示为空

2.css和less的对比

css的生产环境和开发环境均正常

less的生产环境url错误，其余均正常

3.生产环境下css编译后代码和less编译后代码对比

less:

4.React官网/react-app-rewired官网/customize-cra官网的issues查找

未找到相关的问题和解决方案

5.React脚手架版本的变更

尝试使用了v16.8.6的React版本：less编译后正常

使用了最新的16.13.0的React版本：less编译后url路径错误

二、破局

实在不得解，下载了customize-cra v0.9.1版本的源码，找到了addLessLoader方法，发现相对路径的添加主要是依赖于react-scripts中的path.js的servedPath的值，其实这个值就是取自于package.json中的homepage字段

如今没有出现这个相对路径，猜想可能是对应的脚本中的值未取到，查看使用的react-scripts@3.4.0源码，发现paths.js导出的内容中并没有这个值servedPath

同时查看react-scripts@3.0.1源码，成功找到servedPath

到此可能真相呼之欲出了，马上去github上找到相关的更新记录，发现如下内容，在#7259次变更中最终合入v3.4.0版本，私有方法paths.servedPath 和 paths.publicUrl属性被合并成了 paths.publicUrlOrPath

react-scripts更新内容

再去查看最新版customize-cra@1.0.0,发现读取此属性也已经被更改成使用config.output.publicPath

customize-cra@1.0.0中addLessLoader更新内容

三、解决方案

1.将customize-cra升级到1.0.0版本

但是得注意，这个版本addLessLoader的配置属性变更较大，需要如下重写

2.修改customize-cra@0.9.1源码，不建议

3.降低react-scripts版本至3.4.0以下，3.3.1及以下版本

四、结论

1.customize-cra版本与react-script版本存在配套关系，但在官网中并未明细写出，需要谨慎使用，建议使用v1.0.0版本的customize-cra，customize-cra@0.9.0需要搭配react-scripts@3.3.1及以下的使用
2.使用第三方开源组件，解决问题最好的方法还是去查看源码
3.外部组件内的私有对象和方法，建议不要私自调用，难免在版本变更后修改私有对象导致bug的产生

三、React中less的使用

在需要的js或者jsx文件中使用import语法导入，在标签对中使用className属性来进行引用即可

但是要注意生产环境下less最终统一编译成css放到同一个文件中，所以会容易出现样式覆盖的情况，所以务必按照编程规范要求来进行样式命名

四、总结小坑

4.1 npm run eject报错

原理：创建项目时添加了.git文件，但是没有初始化本地仓库

解决方案：

1.删除.git文件

2.将项目关联到代码仓库后再进行(请自行尝试)

4.2 修改配置后less生效了但是注释被编译器当做错误

原理：一个文件被匹配后，会经过多种loader进行转换处理，执行的顺序是从最后一个loader开始向前处理，比如一个css文件就会途径less-loader、postcss-loader、css-loader最终到style-loader

所以如果之前把less-loader加在postcss-loader之前，就会先加载postcss-loader从而判定//注释为错误

解决方案：

将css-loader,less-loader,postcss-loader的顺序改成css-loader,postcss-loader,less-loader
